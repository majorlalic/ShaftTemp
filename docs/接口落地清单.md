# 接口落地清单

## 1. 本次接口落地原则

- 路径统一保留 `/api`
- 响应统一为：
  - `code`
  - `message`
  - `data`
- 字段名以当前数据库和代码为准，不强行对齐文档旧字段
- 告警状态接口同时返回：
  - `statusCode`
  - `statusName`
- 告警处理接口统一为 `action` 风格

## 2. 已确认的业务约定

### 2.1 告警计数字段

告警列表中的：

- `alarmCount`

统一映射为：

- `event_count`

### 2.2 告警分类

同一张 `alarm` 表按 `alarm_type` 区分两类业务：

#### 监测对象报警

- `TEMP_THRESHOLD`
- `TEMP_DIFFERENCE`
- `TEMP_RISE_RATE`

#### 终端报警

- `DEVICE_OFFLINE`
- `PARTITION_FAULT`

### 2.3 告警处理动作

统一动作值：

- `confirm`
- `observe`
- `false_positive`
- `close`

### 2.4 `/monitor/statistics`

该接口含义确认如下：

- 统计不同 `shaft_type` 的告警数量

### 2.5 终端接入状态

复用 `device.asset_status`，建议业务值：

- `PENDING_ACCESS`
  待接入
- `CONNECTED`
  已接入
- `DISABLED`
  停用

## 3. 接口清单（按 Controller 分组）

### 3.1 IoT 服务接收（IotIngestController）

- `POST /api/reports/measure`
- `POST /api/reports/alarm`
- `POST /api/service/iot/data/batch`
- `POST /api/service/iot/alarm/batch`

说明：

- `measure` 驱动实时链路（Redis、`raw_data`、规则、`alarm/event`）
- `alarm` 进入 `alarm_raw`
- `/api/service/iot/*` 为 iot 模块批量回调入口

### 3.2 告警管理（AlarmController）

#### 3.2.1 获取告警状态统计

- `GET /api/alarm/statistics`

说明：

- 统计 `alarm` 表中各状态数量
- 支持按 `areaId / deviceId / startTime / endTime` 过滤

#### 3.2.2 获取告警列表

- `GET /api/alarm/list`

说明：

- 支持分页
- 支持按 `areaId / deviceId / startTime / endTime / status` 过滤
- 返回：
  - 主单信息
  - `alarmCount = event_count`
  - `statusCode + statusName`
  - `areaName / deviceName / deviceType`

#### 3.2.3 获取告警合并事件列表

- `GET /api/alarm/merge/list`

说明：

- 按 `alarmId` 查询 `event` 列表
- 返回分页结构
- 重点字段：
  - `id`
  - `eventTime`
  - `eventContent`

#### 3.2.4 获取告警详情

- `GET /api/alarm/detail?id=...`

说明：

- 返回主单详情
- 带名称类冗余字段：
  - `monitorName`
  - `deviceName`
  - `areaName`

#### 3.2.5 单条处理告警

- `POST /api/alarm/handle`

入参建议：

```json
{
  "id": 1,
  "action": "confirm",
  "handler": 1001,
  "handleRemark": "现场确认"
}
```

#### 3.2.6 批量处理告警

- `POST /api/alarm/batchHandle`

入参建议：

```json
{
  "alarmIds": [1, 2, 3],
  "action": "observe",
  "handler": 1001,
  "handleRemark": "持续观察"
}
```

说明：

- 批量请求默认是同一动作
- 不做事务全成全败
- 返回：
  - `successCount`
  - `failCount`
  - `failIds`

#### 3.2.7 告警规则 CRUD

- `GET /api/alarm-rules`
- `GET /api/alarm-rules/{id}`
- `POST /api/alarm-rules`
- `PUT /api/alarm-rules/{id}`
- `POST /api/alarm-rules/{id}/enable`
- `POST /api/alarm-rules/{id}/disable`
- `DELETE /api/alarm-rules/{id}`

#### 3.2.8 终端告警与事件视图

- `GET /api/terminal/alarm/stat`
- `GET /api/terminal/alarm/list`
- `PUT /api/terminal/alarm/handle`
- `PUT /api/terminal/alarm/batchHandle`
- `GET /api/terminal/alarm/{id}/detail`
- `GET /api/events`

### 3.3 监控对象管理（MonitorController）

#### 3.3.1 获取监测对象详情

- `GET /api/monitor/{id}`

建议返回拆成几部分：

- 基本信息
- 设备信息
- 楼层列表
- 分区绑定列表

这样比一次返回纯平铺字段更好维护。

#### 3.3.2 获取监测对象告警统计

- `GET /api/monitor/statistics`

说明：

- 按 `shaft_type` 统计告警数量

#### 3.3.3 原始数据查询

- `GET /api/raw-data`

说明：

- 当前需传 `from / to`
- 可按 `monitorId / deviceId / shaftFloorId / partitionId` 过滤

### 3.4 设备管理（DeviceController）

#### 3.4.1 获取终端统计信息

- `GET /api/terminal/statistics`

说明：

- 按设备总数、在线数、离线数、在线率统计
- 支持按 `orgId / startTime / endTime`

#### 3.4.2 获取终端详情

- `GET /api/terminal/{id}`

#### 3.4.3 获取终端台账列表

- `GET /api/terminal/ledger/list`

说明：

- 支持分页
- 支持按：
  - `deviceType`
  - `company`
  - `model`
  - `orgId`
  - `startDate`
  - `endDate`

#### 3.4.4 获取终端台账统计

- `GET /api/terminal/stat`

#### 3.4.5 获取终端接入列表

- `GET /api/terminal/access/list`

说明：

- 基于 `device.asset_status`
- `status` 支持：
  - `PENDING_ACCESS`
  - `CONNECTED`
  - `DISABLED`

#### 3.4.6 提交终端接入确认

- `POST /api/terminal/access/confirm`

说明：

- 本质是批量修改 `device.asset_status`
- 同时建立必要的绑定关系

#### 3.4.7 设备实时状态

- `GET /api/devices/{deviceId}/realtime`

### 3.5 area/org（AreaOrgController）

#### 3.5.1 导入组织树数据

- `POST /api/organization/import`

说明：

- 先用 CSV
- 不上 Excel 依赖
- `type` 支持：
  - `org`
  - `area`

#### 3.5.2 获取组织树结构

- `GET /api/organization/tree`

#### 3.5.3 下载导入模板

- `GET /api/organization/template?type=org`
- `GET /api/organization/template?type=area`

说明：

- 返回 CSV 模板文件下载

## 4. 建议实现顺序

建议按以下顺序开发，避免一次铺太开：

1. 响应包装统一
2. 告警接口统一到文档风格
3. 终端告警视图接口
4. 监测对象详情与统计
5. 终端台账与接入接口
6. 规则 CRUD
7. 组织树查询、模板下载、导入
