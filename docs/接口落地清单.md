# 接口落地清单

## 1. 本次接口落地原则

- 路径统一保留 `/api`
- 响应统一为：
  - `code`
  - `message`
  - `data`
- 字段名以当前数据库和代码为准，不强行对齐文档旧字段
- 告警状态接口同时返回：
  - `statusCode`
  - `statusName`
- 告警处理接口统一为 `action` 风格

## 2. 已确认的业务约定

### 2.1 告警计数字段

告警列表中的：

- `alarmCount`

统一映射为：

- `event_count`

### 2.2 告警分类

同一张 `alarm` 表按 `alarm_type` 区分两类业务：

#### 监测对象报警

- `TEMP_THRESHOLD`
- `TEMP_DIFFERENCE`
- `TEMP_RISE_RATE`

#### 终端报警

- `DEVICE_OFFLINE`
- `PARTITION_FAULT`

### 2.3 告警处理动作

统一动作值：

- `confirm`
- `observe`
- `false_positive`
- `close`

### 2.4 `/monitor/statistics`

该接口含义确认如下：

- 统计不同 `shaft_type` 的告警数量

### 2.5 终端接入状态

复用 `device.asset_status`，建议业务值：

- `PENDING_ACCESS`
  待接入
- `CONNECTED`
  已接入
- `DISABLED`
  停用

## 3. 接口清单

## 3.1 告警相关

### 3.1.1 获取告警状态统计

- `GET /api/alarm/statistics`

说明：

- 统计 `alarm` 表中各状态数量
- 支持按 `areaId / deviceId / startTime / endTime` 过滤

### 3.1.2 获取告警列表

- `GET /api/alarm/list`

说明：

- 支持分页
- 支持按 `areaId / deviceId / startTime / endTime / status` 过滤
- 返回：
  - 主单信息
  - `alarmCount = event_count`
  - `statusCode + statusName`
  - `areaName / deviceName / deviceType`

### 3.1.3 获取告警合并事件列表

- `GET /api/alarm/merge/list`

说明：

- 按 `alarmId` 查询 `event` 列表
- 返回分页结构
- 重点字段：
  - `id`
  - `eventTime`
  - `eventContent`

### 3.1.4 获取告警详情

- `GET /api/alarm/detail?id=...`

说明：

- 返回主单详情
- 带名称类冗余字段：
  - `monitorName`
  - `deviceName`
  - `areaName`

### 3.1.5 单条处理告警

- `POST /api/alarm/handle`

入参建议：

```json
{
  "id": 1,
  "action": "confirm",
  "handler": 1001,
  "handleRemark": "现场确认"
}
```

### 3.1.6 批量处理告警

- `POST /api/alarm/batchHandle`

入参建议：

```json
{
  "alarmIds": [1, 2, 3],
  "action": "observe",
  "handler": 1001,
  "handleRemark": "持续观察"
}
```

说明：

- 批量请求默认是同一动作
- 不做事务全成全败
- 返回：
  - `successCount`
  - `failCount`
  - `failIds`

## 3.2 监测对象相关

### 3.2.1 获取监测对象详情

- `GET /api/monitor/{id}`

建议返回拆成几部分：

- 基本信息
- 设备信息
- 楼层列表
- 分区绑定列表

这样比一次返回纯平铺字段更好维护。

### 3.2.2 获取监测对象告警统计

- `GET /api/monitor/statistics`

说明：

- 按 `shaft_type` 统计告警数量

## 3.3 监测对象规则相关

### 3.3.1 告警规则 CRUD

建议接口：

- `GET /api/alarm-rules`
- `GET /api/alarm-rules/{id}`
- `POST /api/alarm-rules`
- `PUT /api/alarm-rules/{id}`
- `POST /api/alarm-rules/{id}/enable`
- `POST /api/alarm-rules/{id}/disable`
- `DELETE /api/alarm-rules/{id}`

## 3.4 终端统计相关

### 3.4.1 获取终端统计信息

- `GET /api/terminal/statistics`

说明：

- 按设备总数、在线数、离线数、在线率统计
- 支持按 `orgId / startTime / endTime`

### 3.4.2 获取终端告警统计

- `GET /api/terminal/alarm/stat`

说明：

- 按终端告警类型统计
- 只看：
  - `DEVICE_OFFLINE`
  - `PARTITION_FAULT`

### 3.4.3 获取终端告警列表

- `GET /api/terminal/alarm/list`

说明：

- 是 `alarm` 表的终端告警视图
- 支持分页和状态筛选

### 3.4.4 处理终端告警

- `PUT /api/terminal/alarm/handle`

说明：

- 与监测告警处理逻辑一致
- 只是限制终端告警类型

### 3.4.5 批量处理终端告警

- `PUT /api/terminal/alarm/batchHandle`

### 3.4.6 获取终端告警详情

- `GET /api/terminal/alarm/{id}/detail`

## 3.5 终端信息相关

### 3.5.1 获取终端详情

- `GET /api/terminal/{id}`

### 3.5.2 获取终端台账列表

- `GET /api/terminal/ledger/list`

说明：

- 支持分页
- 支持按：
  - `deviceType`
  - `company`
  - `model`
  - `orgId`
  - `startDate`
  - `endDate`

### 3.5.3 获取终端台账统计

- `GET /api/terminal/stat`

### 3.5.4 获取终端接入列表

- `GET /api/terminal/access/list`

说明：

- 基于 `device.asset_status`
- `status` 支持：
  - `PENDING_ACCESS`
  - `CONNECTED`
  - `DISABLED`

### 3.5.5 提交终端接入确认

- `POST /api/terminal/access/confirm`

说明：

- 本质是批量修改 `device.asset_status`
- 同时建立必要的绑定关系

## 3.6 组织管理相关

### 3.6.1 导入组织树数据

- `POST /api/organization/import`

说明：

- 先用 CSV
- 不上 Excel 依赖
- `type` 支持：
  - `org`
  - `area`

### 3.6.2 获取组织树结构

- `GET /api/organization/tree`

### 3.6.3 下载导入模板

- `GET /api/organization/template?type=org`
- `GET /api/organization/template?type=area`

说明：

- 返回 CSV 模板文件下载

## 4. 建议实现顺序

建议按以下顺序开发，避免一次铺太开：

1. 响应包装统一
2. 告警接口统一到文档风格
3. 终端告警视图接口
4. 监测对象详情与统计
5. 终端台账与接入接口
6. 规则 CRUD
7. 组织树查询、模板下载、导入
