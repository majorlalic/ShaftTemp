# 人工联调测试流程（空库版）

适用当前 `service` 工程接口：
- `POST /shaft/iot/reports/measure`
- `POST /shaft/iot/reports/alarm`
- `GET /shaft/alarm/list`
- `GET /shaft/alarm/device/alarms`
- `GET /shaft/alarm/merge/list`

默认服务地址：`http://localhost:8080`

## 1. 初始化（空库）

先保证表结构已建好，然后执行：

```sql
source docs/本地联调初始化.sql;
```

该脚本会初始化以下关键主数据：
- 区域/机构：`ODS_DWEQ_DM_AREA_D`、`ODS_DWEQ_DM_ORG_D`
- 设备：`ODS_DWEQ_DM_DEVICE_D`（`iot_code=shaft-dev-01`）
- 监控对象：`ODS_DWEQ_DM_MONITOR_D`（`id=4001`）
- 楼层：`ODS_DWEQ_DM_SHAFT_FLOOR_D`（`5001/5002`）
- 绑定：
  - `ODS_DWEQ_DM_MONITOR_DEVICE_BIND_D`
  - `ODS_DWEQ_DM_MONITOR_PARTITION_BIND_D`
- 规则：`ODS_DWEQ_DM_ALARM_RULE_D`

关键说明：
- 如果缺 `MONITOR_PARTITION_BIND`，推送测温会报：
  - `Partition binding not found for partitionCode=...`

## 2. 基础校验 SQL

```sql
select count(*) as c from ODS_DWEQ_DM_DEVICE_D where deleted=0;

select count(*) as c from ODS_DWEQ_DM_MONITOR_PARTITION_BIND_D where deleted=0 and bind_status=1;
select count(*) as c from ODS_DWEQ_DM_ALARM_RULE_D where deleted=0 and enabled=1;
```

## 3. 推送实时测温（正常值）

请求：

```bash
curl -sS -X POST "http://localhost:8080/shaft/iot/reports/measure" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "/TMP/shaft-dev-01_TMP_th01/Measure",
    "iotCode": "shaft-dev-01",
    "PartitionId": 1,
    "IedFullPath": "/IED/shaft-dev-01",
    "dataReference": "/TMP/shaft-dev-01_TMP_th01",
    "MaxTemp": 55.0,
    "MinTemp": 50.0,
    "AvgTemp": 52.0,
    "MaxTempPosition": 11.0,
    "MinTempPosition": 3.0,
    "MaxTempChannel": 1,
    "MinTempChannel": 1
  }'
```

预期落表：
- `ODS_DWEQ_DM_RAW_DATA_D`：新增 1 条
- `ODS_DWEQ_DM_DEVICE_D`：`online_status=1`，`last_report_time` 更新
- `ODS_DWEQ_DM_DEVICE_ONLINE_LOG_D`：若之前离线，新增上线日志
- `ODS_DWEQ_DM_ALARM_D / ODS_DWEQ_DM_EVENT_D`：此步骤通常不新增

校验 SQL：

```sql
select * from ODS_DWEQ_DM_RAW_DATA_D order by id desc limit 3;
select id, iot_code, online_status, last_report_time from ODS_DWEQ_DM_DEVICE_D where id=3001;
```

from ODS_DWEQ_DM_EVENT_D

## 5. 验证告警合并

同一监测对象同一告警类型会合并（待确认状态）。

重复推送 2~3 次定温：

```bash
curl -sS -X POST "http://localhost:8080/shaft/iot/reports/measure" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "/TMP/shaft-dev-01_TMP_th01/Measure",
    "iotCode": "shaft-dev-01",
    "PartitionId": 1,
    "IedFullPath": "/IED/shaft-dev-01",
    "dataReference": "/TMP/shaft-dev-01_TMP_th01",
    "MaxTemp": 88.0,
    "MinTemp": 60.0,
    "AvgTemp": 74.0
  }'
```

预期：
- `ODS_DWEQ_DM_ALARM_D.merge_count` 增加
- `ODS_DWEQ_DM_ALARM_D.event_count` 增加
- `ODS_DWEQ_DM_EVENT_D` 新增事件（受事件限频配置影响）

查看合并事件接口：

```bash
curl -sS "http://localhost:8080/shaft/alarm/list?pageNo=1&pageSize=20"
curl -sS "http://localhost:8080/shaft/alarm/merge/list?alarmId=<告警ID>&pageNo=1&pageSize=20"
```

## 7. 验证离线巡检告警（DEVICE_OFFLINE）

条件：
- 不再推送测温
- 等待时间 >= `离线阈值 + 巡检间隔`
- 当前初始化规则离线阈值是 `30s`，巡检默认 `30s`

建议等待 70~90 秒后查询：

```sql
select id, alarm_type, status, content, first_alarm_time, last_alarm_time
from ODS_DWEQ_DM_ALARM_D
where deleted=0 and alarm_type='DEVICE_OFFLINE'
order by id desc;
```

  -H "Content-Type: application/json" \

校验 SQL：

```sql
select id, iot_code, topic, partition_id, alarm_status, fault_status, data_reference, collect_time
from ODS_DWEQ_DM_ALARM_RAW_D
order by id desc
limit 20;
```

## 9. 常见问题

1. `Partition binding not found for partitionCode=...`
- 检查 `ODS_DWEQ_DM_MONITOR_PARTITION_BIND_D` 是否有对应 `partition_code` / `data_reference`

2. 推了测温但没告警
- 检查规则表 `ODS_DWEQ_DM_ALARM_RULE_D` 是否启用
- 检查温度是否超过阈值

3. 告警没合并
- 合并仅对“待确认”状态
- 同监测对象 + 同告警类型才会合并

4. 事件条数少于 merge_count
- 受事件限频影响（`shaft.alarm.event-throttle-seconds`）
