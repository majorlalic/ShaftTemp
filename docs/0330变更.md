## 1.topic订阅变化
  - 最新的对接文档参照在docs目录下电缆测温mqtt消息.txt
  - 之前接收的都是单个分区的数据，现在接收的是单个设备下多个分区的数据，而且包装成数组了
  - 数组中每项都添加了partitionId，我觉得area表中如果是楼层级别，应该要有属性字段能对应上partitionId
  - /TMP/xinfu_16A/Measure，xinfu_16A应该能和device表中的iotcode对应上
  - 告警数据topic，不参与我们的所有告警流程，告警topic数据同样需建表存储，将数组拆成行，对象拆成列存储
## 2.不做数据自动删除，暂不考虑数据过大的情况
## 3.实时数据根据“设备唯一标识分区”存储，
- 设备唯一标识就是topic /TMP/xinfu_16A/Measure中xinfu_16A这一段
- 暂不考虑数据量过多的场景，初期上线设备较少，且频率为20分钟一次
## 4.告警添加是否推送的字段，0为为推送，1为已推送
## 5.分页查询的接口格式统一
- 请求参数
```json
{
  "pageNo": 1,
  "pageSize": 10,
  "param1": 1,
  "param2": "frgr"
}
```
- 返回结构

```json
{
  "code": 200,
  "message": "请求成功",
  "success": true,
  "data": {
    "dataList": [],
    "pageNo": 1,
    "totalSize": 12
  }
}
```
