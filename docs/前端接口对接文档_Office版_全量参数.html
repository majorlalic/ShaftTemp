前端接口对接文档（全量参数版）

更新时间：2026-04-08

服务根路径：http://localhost:8090/api。以下路径均需拼接前缀，例如 /shaft/alarm/list 实际请求为 http://localhost:8090/api/shaft/alarm/list。

1. 统一返回结构

除文件下载接口外，统一返回 RestObject<T>：

{
  "code": 200,
  "message": "success",
  "success": true,
  "data": {}
}

分页接口的 data 为 PagePayload<T>：

{
  "total": 100,
  "list": [],
  "pageNo": 1,
  "totalSize": 100,
  "dataList": []
}

告警 action 取值：confirm / observe / false_positive / close

告警 statusCode 取值：0 待确认，1 已确认，2 持续观察，3 误报，4 自动恢复

2. 区域机构接口（AreaOrgController）

2.1 GET /shaft/areaOrg/tree

说明：获取机构树与区域树。

请求参数：无。

data 结构：

{
  "orgTree": [ { "id": 2000001, "name": "机构1", "children": [] } ],
  "areaTree": [ { "id": 1000001, "name": "区域1", "children": [] } ]
}

2.2 GET /shaft/areaOrg/template

说明：下载导入模板。

参数类型必填说明
typestring是org 或 area

返回：文件流，不是 RestObject。

HTTP 200
Content-Type: text/csv;charset=UTF-8
Content-Disposition: attachment; filename="organization-org-template.csv"
Body: CSV 文件字节流

2.3 POST /shaft/areaOrg/import

说明：导入 CSV。

参数位置类型必填说明
typeform-datastring是org 或 area
fileform-datafile是CSV 文件

data 结构：

{
  "total": 100,
  "success": 98,
  "failed": 2,
  "errors": ["第3行：name不能为空"]
}

3. IoT 接口（IotIngestController）

3.1 POST /shaft/iot/reports/measure

说明：单条测温上报，包含实时状态更新、原始数据落库、规则判断、告警事件。

字段类型必填说明
topicstring否MQ topic
iotCodestring否设备编码，建议必传
PartitionIdint否分区ID
IedFullPathstring是完整路径
dataReferencestring是数据引用，例如 /TMP/shaft-dev-001_TMP_th23
MaxTempdecimal是最大温度
MinTempdecimal是最小温度
AvgTempdecimal是平均温度
MaxTempPositiondecimal否最大温位置
MinTempPositiondecimal否最小温位置
MaxTempChannelint否最大温通道
MinTempChannelint否最小温通道
timestampdatetime否采集时间，ISO 格式

data 结构（IngestResult）：

{
  "deviceId": 3000001,
  "monitorId": 4000001,
  "partitionCode": "shaft-dev-001_TMP_th23",
  "rawDataId": 9000001,
  "alarmCount": 1,
  "metrics": { "maxTemp": 78.6, "minTemp": 62.3, "avgTemp": 70.1 }
}

3.2 POST /shaft/iot/reports/alarm

说明：单条告警状态上报，写入 alarm_raw。

字段类型必填说明
topicstring否MQ topic
iotCodestring否设备编码，建议必传
PartitionIdint否分区ID
IedFullPathstring是完整路径
dataReferencestring是数据引用
AlarmStatusboolean是告警状态
FaultStatusboolean是故障状态
timestampdatetime否采集时间

data 结构（IngestResult）：

{
  "deviceId": null,
  "monitorId": null,
  "partitionCode": null,
  "rawDataId": 9100001,
  "alarmCount": 0,
  "metrics": null
}

3.3 POST /shaft/iot/data/batch

说明：批量测温处理，不写 raw_data，仅做处理链路。

请求体：PartitionMeasureRequest 数组（字段同 3.1）。

data 结构：

[
  {
    "deviceId": 3000001,
    "monitorId": 4000001,
    "partitionCode": "shaft-dev-001_TMP_th01",
    "alarmCount": 0
  }
]

3.4 POST /shaft/iot/alarm/batch

说明：批量告警状态上报，当前返回 ack。

请求体：PartitionAlarmRequest 数组（字段同 3.2）。

data 结构：字符串，例如 ack:10。

6. 告警接口（AlarmController）

口径说明：

1) /shaft/alarm/list 通过 type 参数区分告警域。

2) type=0：终端告警（DEVICE_OFFLINE、PARTITION_FAULT）；type=1：监测对象告警（TEMP_THRESHOLD、TEMP_DIFFERENCE、TEMP_RISE_RATE）。

3) /shaft/alarm/device/alarms 仍保留为终端告警专用分页接口。

6.1 GET /shaft/alarm/statistics

用途：告警统计，可按 domain 区分终端/监测对象。

参数类型必填说明
areaIdlong否区域ID
deviceIdlong否设备ID
startTimedatetime否yyyy-MM-dd HH:mm:ss
endTimedatetime否yyyy-MM-dd HH:mm:ss
domainstring否告警域：monitor/device/all（不传等价 all）

data 结构：告警统计对象（total、pendingConfirm、confirmed、observe、falsePositive 等）。

6.2 GET /shaft/alarm/list

用途：告警中心分页查询，可用一个参数区分终端和监测对象。

参数类型必填说明
pageNoint否页码，优先
pageNumint否兼容参数
pageSizeint否每页条数
areaIdlong否区域ID
areaNamestring否小区名称，模糊匹配
deviceIdlong否设备ID
startTimedatetime否yyyy-MM-dd HH:mm:ss
endTimedatetime否yyyy-MM-dd HH:mm:ss
statusstring否状态码 0/1/2/3/4
alarmTypeBigint否告警大类，当前 0=竖井测温
typeint否告警域分组：0=终端告警，1=监测对象告警

data 结构：PagePayload<告警对象>，单条含 id、alarmType、statusCode、firstAlarmTime、lastAlarmTime、mergeCount、eventCount、content、partitionCode、shaftFloorId 等。

6.3 GET /shaft/alarm/merge/list

参数类型必填说明
alarmIdlong是告警ID
pageNoint否页码，优先
pageNumint否兼容参数
pageSizeint是每页条数

data 结构：PagePayload<事件对象>。

6.4 GET /shaft/alarm/detail

参数类型必填说明
idlong是告警ID

data 结构：告警详情对象（字段同告警列表单条，附 handler、handleTime、handleRemark）。

6.5 POST /shaft/alarm/handle

说明：监测对象告警单条处理。

字段类型必填说明
idlong是告警ID
actionstring是confirm/observe/false_positive/close
handlerlong否处理人，confirm 建议传
handleRemarkstring否处理备注

data 结构：处理结果对象（affected、updatedAlarmIds、action）。

6.6 POST /shaft/alarm/batchHandle

说明：监测对象告警批量处理。

字段类型必填说明
alarmIdslong[]是告警ID数组
actionstring是confirm/observe/false_positive/close
handlerlong否处理人
handleRemarkstring否处理备注

data 结构：处理结果对象（affected、updatedAlarmIds、action）。

6.7 GET /shaft/alarm/stat

说明：终端告警统计。

请求参数：无。

data 结构：终端告警统计对象。

6.8 GET /shaft/alarm/device/alarms

用途：终端告警分页查询（仅 DEVICE_OFFLINE / PARTITION_FAULT）。

参数类型必填说明
pageNoint否页码，优先
pageNumint否兼容参数
pageSizeint否每页条数
statusstring否状态码

data 结构：PagePayload<终端告警对象>。

6.9 PUT /shaft/alarm/handle

说明：终端告警单条处理，请求体字段与 6.5 相同。

data 结构：处理结果对象。

6.10 PUT /shaft/alarm/batchHandle

说明：终端告警批量处理，请求体字段与 6.6 相同。

data 结构：处理结果对象。

6.11 GET /shaft/alarm/detail/{id}

路径参数：id（long，必填）。

data 结构：终端告警详情对象。

6.12 GET /shaft/alarm/events

参数类型必填说明
alarmIdlong否告警ID
shaftFloorIdlong否楼层ID
partitionCodestring否分区编码

data 结构：事件列表，单条含 id、alarmId、eventType、eventTime、content、partitionCode、detailJson 等。

6.13 GET /shaft/alarm/alarm-rules

参数类型必填说明
bizTypestring否MONITOR/DEVICE
alarmTypestring否告警类型
scopeTypestring否当前为 GLOBAL
enabledint否0/1

data 结构：规则列表。

6.14 GET /shaft/alarm/alarm-rules/{id}

路径参数：id（long，必填）。

data 结构：规则详情对象（ruleName、bizType、alarmType、thresholdValue、enabled 等）。

6.15 POST /shaft/alarm/alarm-rules

说明：新增规则。

字段类型必填说明
ruleNamestring是规则名
bizTypestring是MONITOR/DEVICE
alarmTypestring是告警类型
scopeTypestring是GLOBAL
scopeIdlong否全局规则可空
levelint否等级
thresholdValuedecimal否阈值1
thresholdValue2decimal否阈值2
durationSecondsint否持续时长秒
enabledint否0/1
remarkstring否备注

data 结构：新增后的规则对象。

6.16 PUT /shaft/alarm/alarm-rules/{id}

说明：修改规则。路径参数 id（long，必填）。请求体字段同 6.15。

data 结构：更新后的规则对象。

6.17 POST /shaft/alarm/alarm-rules/{id}/enable

路径参数：id（long，必填）。

data 结构：操作结果对象（id、enabled）。

6.18 POST /shaft/alarm/alarm-rules/{id}/disable

路径参数：id（long，必填）。

data 结构：操作结果对象（id、enabled）。

6.19 DELETE /shaft/alarm/alarm-rules/{id}

路径参数：id（long，必填）。

data 结构：删除结果对象（id、deleted）。

7. 联调建议

1) 先导入“达梦联调初始化_千级数据.sql”。

2) 再用 Postman 集合按顺序执行：IoT 上报 -> 告警查询 -> 事件查询。

3) 分区编码必须大小写一致，建议使用示例：shaft-dev-001_TMP_th23。
