# 前端接口对接文档（最新）

更新时间：2026-04-07  
来源：`service/src/main/java/com/example/demo/controller/*` 实际映射

## 1. 通用约定

- 服务基准路径：无统一前缀，按各 Controller 的 `@RequestMapping` 使用。
- 默认返回结构（除文件下载接口）：

```json
{
  "code": 200,
  "message": "success",
  "success": true,
  "data": {}
}
```

- 分页结构（`data` 为 `PagePayload`）：

```json
{
  "total": 100,
  "list": [],
  "pageNo": 1,
  "totalSize": 100,
  "dataList": []
}
```

- 告警处理动作 `action` 可选值：
`confirm` / `observe` / `false_positive` / `close`

- 告警状态 `statusCode`：
`0` 待确认，`1` 已确认，`2` 持续观察，`3` 误报，`4` 自动恢复

| GET | `/shaft/alarm/merge/list` | 告警合并事件分页 |
| GET | `/shaft/alarm/detail` | 告警详情（query 版） |
| POST | `/shaft/alarm/handle` | 单条处理（监测对象） |
| POST | `/shaft/alarm/batchHandle` | 批量处理（监测对象） |
| GET | `/shaft/alarm/stat` | 终端告警统计 |
| GET | `/shaft/alarm/device/alarms` | 终端告警分页 |
| PUT | `/shaft/alarm/handle` | 单条处理（终端） |
| PUT | `/shaft/alarm/batchHandle` | 批量处理（终端） |
| GET | `/shaft/alarm/detail/{id}` | 终端告警详情（path 版） |
| GET | `/shaft/alarm/events` | 事件列表 |
| GET | `/shaft/alarm/alarm-rules` | 规则列表 |

| POST | `/shaft/alarm/alarm-rules/{id}/disable` | 停用规则 |
| DELETE | `/shaft/alarm/alarm-rules/{id}` | 删除规则（逻辑删） |

### 2.1 查询参数说明（常用）

- `/statistics`：`areaId`、`deviceId`、`startTime`、`endTime`
- `/list`：`pageNo/pageNum`、`pageSize`、`areaId`、`deviceId`、`startTime`、`endTime`、`status`
- `/merge/list`：`alarmId`、`pageNo/pageNum`、`pageSize`
- `/events`：`alarmId`、`shaftFloorId`、`partitionCode`
- 时间格式：`yyyy-MM-dd HH:mm:ss`

### 2.2 请求体说明（处理类）

`POST/PUT /shaft/alarm/handle`

```json
{
  "id": 1770000000000001,
  "action": "confirm",
  "handler": 10001,
  "handleRemark": "已确认"
}
```

`POST/PUT /shaft/alarm/batchHandle`

```json
{
  "alarmIds": [1770000000000001, 1770000000000002],
  "action": "false_positive",
  "handler": 10001,
  "handleRemark": "批量处理"
}
```

### 2.3 规则请求体

`POST/PUT /shaft/alarm/alarm-rules[/{id}]`

```json
{
  "ruleName": "温度阈值规则",
  "bizType": "MONITOR",
  "alarmType": "TEMP_THRESHOLD",
  "scopeType": "GLOBAL",
  "scopeId": null,
  "level": 2,
  "thresholdValue": 70.0,
  "thresholdValue2": null,
  "durationSeconds": null,
  "enabled": 1,
  "remark": "默认规则"
}
```

说明：后端会强制按全局规则保存（`scopeType=GLOBAL`，`scopeId=null`）。

## 5. 区域机构模块 `/shaft/areaOrg`

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/shaft/areaOrg/tree` | 机构/区域树 |
| GET | `/shaft/areaOrg/template` | 导入模板下载（CSV文件流） |
| POST | `/shaft/areaOrg/import` | 机构/区域导入 |

### 5.1 模板下载

- 请求：`GET /shaft/areaOrg/template?type=org` 或 `type=area`
- 返回：`text/csv` 文件流（`Content-Disposition: attachment`）
- 文件名：`organization-{type}-template.csv`

### 5.2 导入接口

- `multipart/form-data`
- 字段：
  - `type`：`org` 或 `area`
  - `file`：CSV 文件

## 7. 前端联调建议

- 优先使用 `pageNo` 参数；后端兼容 `pageNum`。
- 时间参数按接口定义传：
  - 统计类常用 `yyyy-MM-dd HH:mm:ss`
  - `raw-data` 用 ISO 时间
- 下载模板接口不要按 JSON 解析，要按文件流（blob）处理。
