1. 表名规范
- 所有表名需要按照ODS_DWEQ_DM_*_D的规范创建
- 使用的是达梦的数据库，需要注意表名超长的情况，过长的表名可适当改名

2. 移除raw_data月表逻辑，保持一张表

3. 移除jdbc的插入

4. 移植的项目分两个模块，iot和service模块

iot模块
接收mqtt数据，对应topic有对应的handler类，因此将有DataHandler处理实时数据topic，AlarmHandler处理告警数据topic
Handler的核心方法如下
```
public void update(String jsonData, String topic);
```

DataHandler和AlarmHandler均需要在update方法中实现以下功能:
- 需解析jsonData数据成vo对象， 我们解析出来的应该都是List<*VO>吧？
- 实现对raw_data和alarm_raw的存储
- 需通过http接口调用service模块中的接口传输vo对象到service模块

-------- 

service模块
- 提供接口接收iot模块的vo对象
- 按理说AlarmHandler调用的接口是不需要做什么事了，可以空着，因为原始告警数据已经在iot模块存储了
- DataHandler调用的接口则需要处理之前告警生成，合并的逻辑
- 告警规则的维护和各类之前说的查询接口也需要在这个模块编写，可能还有那些漏的你帮我想想


- 持久化service，dao


5. 题外话
- 使用的是达梦数据库，数据库脚本和初始化脚本兼容
- 用mybatis做持久层
- 我希望项目目录结构不要太多，controller，service，dao，vo，entity等包即可，复杂的合并告警等逻辑都可以在service中处理
